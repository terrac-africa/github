1 Environment Setup

Configure your local Git identity. This is required for your commits to be attributed correctly.

git config --global user.name "Your Name"
git config --global user.email "your.email@terrac.co.za"
git config --list

2 Init & Commit

Create a new directory and initialize it as a Git repository.

mkdir my-project
cd my-project
git init
echo "# My Project" >> README.md
git add README.md
git commit -m "First commit"

3 Branching Strategy

Never commit directly to main. Create a feature branch.

git checkout -b feature/update-docs
# Make some changes to README.md
git status
git add .
git commit -m "Updated documentation"
git push origin feature/update-docs
